Vite 多页面配置

问题背景： 针对使用 vue3 开发的 h5 应用，根据传统的 vite 启动的单页面应用，一般是一个入口对应一个出口，但 h5 应用慢慢增多，如果还是按照单入口单出口的模式进行开发，意味着每次新增一个 h5 应用的时候，我们需要新搭建一个项目，封装的公共方法、项目的配置也都需要再次进行一次，会产生大量重复工作。

解决思路： 把 vite 的单入口单出口配置改为单入口多出口，按需打包多页面.(为什么不改成多入口多出口呢，因为我们目前涉及到的项目大多为比较简单的 h5 应用，没有必要给每个包配置单独的入口，等后续如果有需求，可以在这个基础上进行改动)。

2、vite.config.ts​

import { fileURLToPath, URL } from "node:url";

import { defineConfig } from "vite";
import { viteMockServe } from "vite-plugin-mock";
import vue from "@vitejs/plugin-vue";
import vueJsx from "@vitejs/plugin-vue-jsx";
import path from "path";
import legacy from '@vitejs/plugin-legacy';

export default defineConfig(({command, mode})=>{
  return {
    plugins: [
      vue(),
      vueJsx(),
      viteMockServe({
        mockPath: "./mock",  // mock文件存放的位置
        localEnabled: command === "serve" && mode === "mock", //在开发环境中启用 mock
      }),
      legacy({
        targets: ['Chrome 52'],
        additionalLegacyPolyfills: ['regenerator-runtime/runtime'],
        modernPolyfills: true,
      }),
    ],
    base: '/app/mes/',
    resolve: {
      alias: {
        "@": fileURLToPath(new URL("./src", import.meta.url)),
      },
    },
    server: {
      host: '0.0.0.0',
      proxy: {
        '^/MESWebApi': {
          target: 'http://172.16.7.181',
          changeOrigin: true, //开启代理
        }
      }
    },
    build: {
      // 浏览器兼容性 ‘esnext’ | 'modules'
      target: "modules",
      //输出路径
      outDir: `${process.cwd()}/dist/${process.argv[process.argv.length-1]}`,
      // 小于此阈值的导入或引用资源将内联为 base64 编码， 以避免额外的http请求， 设置为 0, 可以完全禁用此项，
      assetsInlineLimit: 4096,
      // 启动 / 禁用 CSS 代码拆分
      cssCodeSplit: true,
      // 构建后是否生成 soutce map 文件
      sourcemap: false,
      // 自定义底层的 Rollup 打包配置
      rollupOptions: {
        input: path.resolve(`index.html`),
        output: {
          chunkFileNames: `static/js/[name]-[hash].js`,
          entryFileNames: `static/js/[name]-[hash].js`,
          assetFileNames: `static/[ext]/[name]-[hash].[ext]`,
        },
      },
    },
  }
});

3、打包工具函数 build.js，按需打包​

const { resolve } = require("path");
const inquirer = require("inquirer");
const fs = require('fs');
const child_process= require("child_process");
const chalk = require("chalk");

class Build {
  async main() {
    // 读取配置
    const moduleFile = resolve('.modulerc.ts');
    let moduleConfig = [];
    if (fs.existsSync(moduleFile)) {
      const config = require(moduleFile);
      moduleConfig = config.packages;
    }
    if(process.argv[2]) {
      const page = process.argv[2].split(',');
      await this.handleBuild(page);
      return;
    }
    if(moduleConfig?.length) {
      const choices = moduleConfig.map((item) => item.name);
      this.questions = [
        {
          type: 'checkbox',
          name: "page",
          message: "选择要打包的项目",
          choices,
        },
      ];
    }
    // 命令行交互打包
    await this.selectBuildName();
  }
  
  selectBuildName() {
    return inquirer.prompt(this.questions).then(async (answers) => {
      const { page } = answers;
      await this.handleBuild(page)
    });
  }

  async handleBuild(page) {
    if(page.length) {
      const cmd = `vite build --`;
      // 循环打包
      for (const name of page) {
        try {
          console.log(`-------------------- ${name} 开始打包 --------------------`);
          await new Promise((resolve, reject) => {
            child_process.exec(
              `${cmd} ${name}`,{
                cwd: process.cwd(),
                env: {
                  ...process.env,
                  VITE_APP_TARGET: name,
                },
              },
              (error, stdout, stderr) => {
                console.log('stdout: ' + stdout);
                if (error) {
                  console.log(chalk.red(`${name} 打包失败`));
                  console.error(chalk.red('error:', error));
                  console.log('stderr: ' + stderr);
                  reject()
                  process.exit(1);
                }else {
                  console.log(chalk.green(`${name} 打包成功`));
                  resolve()
                }
              }
            );
          })
        } catch (e) {
          console.log(chalk.red(`${name} 打包失败`));
          console.error(chalk.red('error:', e))
          process.exit(1);
        }
      }
    }
  }
}

const build = new Build();
build.main()

5、router/index.ts 文件​

使用统一入口，根据启动环境加载不同路由文件：

  • 本地启动时加载 applications，提供统一入口，集成所有子包的router，方便开发调试；
  • 生产环境打包时根据 VITE_APP_TARGET 加载对应子包下的路由文件。 
import { createRouter, createWebHashHistory } from "vue-router";
import type { ModuleNamespace } from "vite/types/hot";

const components:any[] = [];

// 获取模块下的所有路由配置
const allModules:Record<string, ModuleNamespace> = import.meta.glob('@/modules/*/router/index.ts', { eager: true });
Object.values(allModules).forEach((item) => {
  components.push(item.default);
});
const routes:any[] = [];
const {DEV:isDev, VITE_APP_TARGET} = import.meta.env

// 开发环境启动时加载applications
if(isDev) {
  routes.push({
    path: '/applications',
    name: 'applications',
    component: ()=>import('../applications.vue'),
  });
  components.forEach((item) => {
    routes.push(...item);
  });
} else {
  // 打包时根据 VITE_APP_TARGET 加载对应路由
  const keys=Object.keys(allModules);
  for(let i=0; i<keys.length; i++) {
    if(keys[i].includes(`/modules/${VITE_APP_TARGET}/router`)) {
      const moduleRoute=allModules[keys[i]].default;
      routes.push(...moduleRoute)
      break;
    }
  }
}
const router = createRouter({
  history: createWebHashHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: '/',
      redirect: routes[0]?.path,
      // redirect: '/loading/1/1/1'
    },
    ...routes,
  ],
});

export default router;

8、单入口总结​

借鉴了 monorepo 的思想，但考虑到我们大多数 H5 页面并不会有太多代码，所以没有做的很复杂（所以没有做这部分：公共子包、子包有自己的配置文件/入口文件，可以单独启动、通过软链进行模块间通信）

分包依据：子包应该包含的部分：页面文件、路由文件、该子包代码里特有的方法/组件等；公共部分 index.html 入口文件、环境变量、公共组件/方法、各种配置文件（后续可以根据需求再做调整）

9、更改为多入口​

原因：vite中通过变量写就会加载所有的文件，并不会根据变量进行拆分：@/modules/${VITE_APP_TARGET}/router/index.ts，打包后会包含所有包的文件 这样写就只加载对应包下的：'@/modules/mes/router/index.ts'

考虑到这个问题，改为多入口，每个包有自己的router文件后，不存在通过变量引入。

  • 修改vite.config.ts如下
build: {
  // 浏览器兼容性 ‘esnext’ | 'modules'
  target: "modules",
  //输出路径
  outDir: `${process.cwd()}/dist/${packageName}`,
  // 小于此阈值的导入或引用资源将内联为 base64 编码， 以避免额外的http请求， 设置为 0, 可以完全禁用此项，
  assetsInlineLimit: 4096,
  // 启动 / 禁用 CSS 代码拆分
  cssCodeSplit: true,
  // 构建后是否生成 soutce map 文件
  sourcemap: false,
  // 自定义底层的 Rollup 打包配置
  rollupOptions: {
    input: path.resolve(`src/modules/${packageName}/index.html`),
    output: {
      chunkFileNames: `static/js/[name]-[hash].js`,
      entryFileNames: `static/js/[name]-[hash].js`,
      assetFileNames: `static/[ext]/[name]-[hash].[ext]`,
    },
  },
},

10、增加新建脚本​

命令：npm run create xxx

create.js

const { mkdir, readFile, writeFile } = require("fs/promises");
const { resolve } = require("path");

const chalk = require("chalk");
const fs = require("fs");

class CreatePages {
  constructor() {
    this.pagesPath = resolve("src/modules");
    this.pageName = process.argv[2];
    this.mkPageDir();
  }
  // 创建目录
  async mkPageDir() {
    if(!this.pageName) return this.log(["red", `未输入创建项目的名称`]);
    try{
      await mkdir(`${this.pagesPath}/${this.pageName}`)
      await mkdir(`${this.pagesPath}/${this.pageName}/router`)
      await mkdir(`${this.pagesPath}/${this.pageName}/views`)
      this.log(["green", `项目创建成功！`]);
      this.createFile();
      this.handleAddModulerc();
    } catch(e) {
      this.log(["red", e.message]);
    }
  }
  // 复制文件
  createFile() {
    ["main.ts", "index.html", "router.ts", "router/index.ts", "views/demo.vue"].forEach(async (item) => {
      try {
        const fileStr = await readFile(
          `./scripts/templates/${item}`,
          "utf-8"
        );
        await writeFile(`${this.pagesPath}/${this.pageName}/${item}`, fileStr, "utf-8");
        this.log(["green", `${item} 文件创建成功！`]);
      } catch (e) {
        this.log(["red", e.message]);
      }
    });
  }
  // 插入.modulerc.ts
  handleAddModulerc() {
    const moduleFile = resolve('.modulerc.json');
    const jsonData = fs.readFileSync(moduleFile)
    const jsonObj=JSON.parse(jsonData);
    jsonObj.push({name: this.pageName})
    fs.writeFileSync(moduleFile, JSON.stringify(jsonObj, null, 2))
  }
  log([key, value]) {
    try {
      console.log(chalk[key](`\n => ${value}`));
    } catch (e) {
      console.log(chalk.red(`\n => ${e.message} \n`));
    }
  }
}

new CreatePages();
