开发常见问题总结

1、关于 vite 的环境变量​

默认环境变量

import.meta.env.MODE: {string} 应用运行的模式 development|production。
import.meta.env.BASE_URL: {string} 部署应用时的基本 URL。他由base配置项决定。
import.meta.env.PROD: {boolean} 应用是否运行在生产环境。
import.meta.env.DEV: {boolean} 应用是否运行在开发环境 (永远与 import.meta.env.PROD相反)。
import.meta.env.SSR: {boolean} 应用是否运行在 server 上。

可以通过传递 --mode 选项标志来覆盖命令使用的默认模式。例如，如果你想为我们假设的 staging 模式构建应用：

vite build --mode staging

为了使应用实现预期行为，我们还需要一个 .env.staging 文件：

# .env.staging
NODE_ENV=production
VITE_APP_TITLE=My App (staging)

自定义环境变量 命名必须以 VITE_ 打头；

4、本地启动 vite 项目，低版本浏览器现实空白​

原因：低版本浏览器无法识别 es6，vite 提供解决的打包插件 legacy；

vite.config.js

import legacy from '@vitejs/plugin-legacy';

export default defineConfig(({command, mode})=>{
  return {
    plugins: [
      legacy({
        targets: ['Chrome 52'],
        additionalLegacyPolyfills: ['regenerator-runtime/runtime'],
        modernPolyfills: true,
      }),
    ],
  }
});

不过这个只解决了打包的问题，本地启动的依旧没有解决

5、跨域问题​

本地通过 vite 配置 proxy 解决跨域​

vite.config.js

server: {
  host: '0.0.0.0',
  cors: true,
  port: 8000,
  open: true,
  proxy: {
    '^/MESWebApi': { // 接口统一前缀
      target: 'http://172.16.7.181',
      changeOrigin: true, //开启代理
    },
    // 如果接口没有统一前缀，可以这样写，给axios的全局配置添加baseUrl为'/api'
    '^/api': { 
      target: 'http://172.16.7.181',
      changeOrigin: true, //开启代理
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}

线上通过 nginx 反向代理解决跨域​

反向代理，跨域：https://cloud.tencent.com/developer/article/1942705

location /app/mes {
  alias /app/mes/;
  error_page 404 /app/mes/index.html;
  index index.html index.htm;
}

location ^~ /MESWebApi {  # mes接口转发
  proxy_pass  http://172.16.7.181/MESWebApi/;

  proxy_set_header Host $host; 

  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header REMOTE-HOST $remote_addr;
}

6、手机端预览本地启动的项目：​

  • 打开cmd，输入ipconfig，得到本机的ip地址
  • 保持电脑和手机在同一局域网，或者电脑给手机开热点
  • 启动命令改为，vite --host xx.xx.xx.xx(本机ip)

7、本地启动dist项目​

  • 全局安装 http-server
  • 在 dist 文件夹下执行 http-server
  • http-server -c-1 命令可以禁止缓存

8、vscode 无法使用 http-server​

  • 使用管理员权限打开工具软件（如vscode）
  • 命令行执行：get-ExecutionPolicy，得到结果为Restricted，表示禁止状态
  • 命令行执行：set-ExecutionPolicy RemoteSigned
  • 再次执行命令：get-ExecutionPolicy，结果显示为RemoteSigned
  • 执行http-server,运行成功

9、拖拽卡顿的问题​

因为touch有穿透效果，在拖拽的时候触发了蒙层下面的滑动事件，导致计算量变大，非常卡顿；

解决办法：@click.stop.prevent=""，stop阻止冒泡，prevent阻止默认事件-滑动

10、输入框判断输入的是中文还是英文​

关键事件：compositionstart 和 compositionend;

<Search
  placeholder="搜索公司名称"
  v-model="searchValue"
  @compositionstart="handleCompositionstart"
  @compositionend="handleCompositionend"
  @input="handleInput"
  @search="handleSearch"
/>
const handleCompositionstart = (e) => {
  // 输入汉语拼音时锁定
  inputlock=true
}

const handleCompositionend = (e) => {
  // 输入汉语拼音时锁定，搜索中文
  inputlock=false
}

const handleInput = (e) => {
  if(!inputlock) {
    // 搜索字母
    console.log('input', searchValue.value)
  }
}

11、微信小程序兼容性问题​

  • 1、background-image 不支持本地图片
  • 2、小程序内部没有 Image 类型；
  • 3、小程序内部的 canvas 不能绘制 svg 格式的图片，不识别

15、github git push时报Failed to connect to github.com port 443 : Timed out​

问题​

Git 所设端口与系统代理不一致，需重新设置

解决​

打开代理：设置 -- 网络与internet -- 查找代理；记录下当前系统代理的 IP 地址和端口号

修改 Git 的网络设置 

# 注意修改成自己的IP和端口号
git config --global http.proxy http://127.0.0.1:7890 
git config --global https.proxy http://127.0.0.1:7890

16、使用 gitlab 报错 502，但服务器是正常的​

承接第15条，这个报错是因为使用了代理，需要取消代理。

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy

# 查看代理
git config --global --get http.proxy
git config --global --get https.proxy
